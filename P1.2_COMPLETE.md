## What Was Implemented

### 1. RabbitMQ Consumer Loop
**File**: `src/rabbitmq_consumer.c`

**Functions**:
- `setup_amqp_connection()` — Establish AMQP connection with TLS support
- `rabbitmq_consumer_create()` — Initialize consumer and connect
- `rabbitmq_consumer_run()` — Main blocking loop to consume messages
- `rabbitmq_consumer_is_connected()` — Check connection status
- `rabbitmq_consumer_shutdown()` — Graceful shutdown signal
- `rabbitmq_consumer_free()` — Cleanup and close connection

**Key Features**:
- ✅ AMQP/TCP connection with credentials from config
- ✅ Channel setup with QoS prefetch=1
- ✅ Parse JSON messages from `ls.commands` queue
- ✅ Message validation (required fields)
- ✅ ACK on success, NACK with requeue on error
- ✅ Proper error handling and cleanup
- ✅ Integration with controller for event routing

### 2. Event Routing Controller
**File**: `src/ls_controller.c`

**Function**: `controller_handle_event()`

**Routes**:
```
control.skip         → "next"
control.shutdown     → "shutdown"
announcement.push    → "announcements.push <filepath>"
```

**Features**:
- ✅ Event type matching (switch on event string)
- ✅ Payload parsing (JSON object extraction)
- ✅ Command validation (safe argument checks)
- ✅ Liquidsoap socket communication
- ✅ Response handling (ok/error)
- ✅ Health tracking (marks unhealthy on socket failure)
- ✅ Proper result codes (RESULT_OK, RESULT_ERROR, RESULT_RETRY)

### 3. Message Processing Flow

```
RabbitMQ
    ↓
rabbitmq_consumer_run() — blocking loop
    ↓
message_parse() — JSON validation
    ↓
message_is_valid() — required fields check
    ↓
controller_handle_event() — route & execute
    ↓
ls_send_command() — TCP to Liquidsoap
    ↓
ACK (success) or NACK (failure)
```

```bash

```
Should show: `✓ Testes OK`

### Integration Test (requires RabbitMQ + Liquidsoap)
1. Start RabbitMQ: `sudo systemctl start rabbitmq-server`
2. Start Liquidsoap: `liquidsoap "exit"` (or run your streams)
3. Configure env vars: `source conf/memphis.env`
4. Run Memphis: `./build/bin/memphis`
5. Publish test message to RabbitMQ:
```bash
rabbitmqctl publish_message radio.events "control.skip" '{"version":1,"id":"test1","timestamp":"2026-04-20T10:00:00Z","event":"control.skip"}'
```
6. Verify Liquidsoap received `next` command

## Known Limitations

1. **No TLS yet** — amqp_tcp_socket (not amqp_ssl_socket)
   - Fix: Use RABBITMQ_TLS_ENABLED=1 and amqp_ssl_socket when available

2. **No message ordering guarantee** — AMQP doesn't guarantee order for fast consumers
   - Fix: Use single-consumer queue or message versioning

3. **No heartbeat** — Uses default (0)
   - Fix: Set AMQP_DEFAULT_HEARTBEAT in code

4. **No automatic reconnect backoff** — Immediate reconnect attempt
   - Fix: Add exponential backoff in consumer_run loop
